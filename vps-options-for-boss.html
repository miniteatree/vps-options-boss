VPS Options for CodeIgniter Frontend / Full Migration

A practical comparison for moving the current CodeIgniter system to VPS in stages. The goal is to keep cost controlled, handle up to roughly 200GB of images, and leave room for future growth without forcing a risky full migration on day one.

Current System
CodeIgniter 3
Image Volume
~200GB
Recommended Path
C2 + Cloudflare R2

Frontend First

Lowest-risk rollout by moving only the public-facing site to VPS first.

Admin Can Stay

Admin panel and backend operations can remain on cPanel temporarily.

Storage Matters

Large image storage should ideally be separated from VPS local disk.

Future-Safe

Object storage gives a cleaner path for scale and later migration.

Current Situation

Key constraints affecting the hosting decision.

  • Current system is built on CodeIgniter 3.
  • Images are currently stored locally and referenced through database paths.
  • Total image storage can reach around 200GB.
  • A staged migration is possible: frontend on VPS first, while admin remains on cPanel.

Detailed Options

Practical pros, tradeoffs, and where each option fits.

Option 1: NVMe C2

Best for a low-risk frontend-first move.

RM60 / month
100GB NVMe · 4GB RAM · 2 vCPU
Frontend on VPS Admin stays on cPanel Fastest to deploy
  • Move only the user-facing frontend to VPS.
  • Images can still be served from cPanel via img.domain.com.
  • Suitable when the immediate goal is speed and minimal change.
Pros
  • Cheapest entry point
  • Low migration risk
  • Quick deployment
Cons
  • Still depends on cPanel
  • Storage architecture remains split

Option 2: NVMe C2 + Cloudflare R2

Strong balance between cost, flexibility, and future growth.

~RM73 / month
RM60 VPS + ~RM13 storage @ 200GB
Object storage Scalable image delivery Cleaner future migration path
  • VPS runs the frontend, while large images move to object storage.
  • Estimated R2 cost: USD0.015 / GB-month.
  • At 200GB, storage is about USD3 / month (~RM13).
  • At 300GB, total estimated cost is around ~RM79 / month.
Pros
  • Does not fill VPS disk with images
  • Good fit for public image delivery
  • More scalable than local-only storage
Cons
  • Needs object storage integration
  • Image URLs may need adjustment

Option 2B: NVMe C2 + Backblaze B2

Very attractive on storage price alone.

~RM65 / month
RM60 VPS + ~RM4–5 storage
Low storage cost Worth evaluating
  • Public pricing found: USD5 / TB / month.
  • Estimated storage cost for 200GB is around RM4–5 / month.
Pros
  • Very low storage cost
Cons
  • Egress / traffic terms should be checked carefully
  • Not the safest default for public-heavy image delivery without review

Option 2C: NVMe C2 + Wasabi

Simple S3-compatible alternative with affordable pricing.

~RM66 / month
RM60 VPS + ~RM6 storage
S3-compatible Affordable
  • Public pricing found: USD6.99 / TB / month.
  • Estimated storage cost for 200GB is around RM6 / month.
Pros
  • Cheap and workable
  • Easy fit if S3 compatibility is preferred
Cons
  • Billing details / minimums should be verified before committing

Option 3: NVMe C6

For full migration when centralization matters more than cost.

RM222 / month
400GB NVMe · 16GB RAM · 8 vCPU
Full migration All-in-one VPS Highest monthly cost
  • Move frontend, admin, database, and images fully to VPS.
  • Architecture becomes simpler after migration because everything is centralized.
Pros
  • Single environment to manage
  • Cleaner stack after full migration
Cons
  • Much higher monthly cost
  • Local VPS disk can still become a limit as image storage grows

Recommendation

Best balance for now: NVMe C2 + Cloudflare R2. This keeps VPS cost low, avoids storing a large image library on local VPS disk, and gives a more future-proof setup if the system grows later.

  • Low monthly VPS cost
  • Images do not consume VPS local storage
  • Better long-term scalability
  • Allows frontend-first migration without forcing immediate admin migration
