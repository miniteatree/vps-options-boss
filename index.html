VPS Options for CodeIgniter Frontend / Full Migration

Prepared for internal review. Pricing based on Exabytes NVMe VPS page and public object storage pricing pages checked today.

Current Situation

  • Current system: CodeIgniter 3
  • Images stored locally, with database path records
  • Image size can reach around 200GB
  • Possible staged migration: frontend on VPS first, admin remains on cPanel

Options Summary

Option Use Case Monthly Cost Notes
NVMe C2 Frontend only on VPS RM60 / month Best if images still remain on cPanel subdomain temporarily.
NVMe C2 + Cloudflare R2 Frontend on VPS + object storage for images ~RM73 / month (for 200GB) Recommended balance of cost, flexibility, and future scaling.
NVMe C2 + Backblaze B2 Frontend on VPS + cheaper object storage ~RM65 / month (for 200GB) Lower storage cost, but public image traffic / egress needs extra care.
NVMe C2 + Wasabi Frontend on VPS + S3-compatible storage ~RM66 / month (for 200GB) Cheap and workable, but should verify billing rules before committing.
NVMe C6 Full migration to VPS RM222 / month Frontend, admin, database, and images all moved to VPS.

Option 1: NVMe C2 (Frontend Only)

  • Price: RM60 / month
  • Specs: 100GB NVMe, 4GB RAM, 2 vCPU
  • Use case: only move user-facing frontend to VPS
  • Admin stays on cPanel
  • Images can continue to be served from cPanel via subdomain such as img.domain.com

Pros: cheapest entry point, low migration risk, fast to deploy.

Cons: still depends on cPanel for image delivery and possibly admin/database.

Option 2: NVMe C2 + Cloudflare R2 (Recommended)

  • VPS: RM60 / month
  • R2 storage: USD0.015 / GB-month
  • Estimated for 200GB: about USD3 / month (~RM13)
  • Total estimated: ~RM73 / month
  • If 300GB: about ~RM79 / month

Pros: suitable for public image delivery, scalable, and avoids filling VPS disk with large image storage.

Cons: requires object storage integration and image URL adjustment.

Option 2B: NVMe C2 + Backblaze B2

  • VPS: RM60 / month
  • Storage pricing found: USD5 / TB / month
  • Estimated for 200GB: about RM4–5 / month
  • Total estimated: ~RM65 / month

Pros: very low storage cost.

Cons: image traffic / egress terms should be checked carefully before using for public frontend image serving.

Option 2C: NVMe C2 + Wasabi

  • VPS: RM60 / month
  • Storage pricing found: USD6.99 / TB / month
  • Estimated for 200GB: about RM6 / month
  • Total estimated: ~RM66 / month

Pros: affordable S3-compatible storage.

Cons: billing details / minimums should be verified first.

Option 3: NVMe C6 (Full Migration)

  • Price: RM222 / month
  • Specs: 400GB NVMe, 16GB RAM, 8 vCPU
  • Use case: move frontend, admin, database, and images fully to VPS

Pros: all systems centralized in one VPS, simpler architecture after migration.

Cons: much higher monthly cost, and local VPS disk may still become limiting if image storage keeps growing.

Recommendation

Best balance for now: NVMe C2 + Cloudflare R2.

  • Low VPS cost
  • Images do not consume VPS local disk
  • Good path for future growth
  • Can migrate frontend first without forcing full admin migration immediately
